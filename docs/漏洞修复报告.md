## 2. 高危漏洞详解

### V-001：密码明文存储 🔴

**位置：** `app.py` 第 9-20 行，`USERS` 字典

**问题代码：**
```python
USERS = {
    "admin": {
        "password": "admin123",  # ← 明文存储
    },
    "alice": {
        "password": "alice2025",  # ← 明文存储
    }
}
```

**风险：**
- 数据库泄露 = 所有密码直接暴露
- 用户习惯复用密码，可导致撞库攻击
- 内部人员可直接获取所有用户密码

**利用方式：** 攻击者若获取源代码或通过任意文件读取漏洞访问到 USERS 字典，即可直接获得所有用户的明文密码。

---

### V-002：密码明文显示 🔴

**位置：** `app.py` 第 31 行 & `templates/index.html` 第 16 行

**问题代码：**
```python
return render_template("index.html", user=user)  # 完整信息（含密码）传给模板
```

```html
<li><span class="user-info-label">密码</span><span class="user-info-value">{{ user['password'] }}</span></li>
```

**风险：**
- 登录后密码直接展示在页面上
- 旁人路过屏幕即可看到密码
- 违反《网络安全法》和等保要求

**利用方式：** 任何能访问到首页的用户都能看到自己的密码（以及管理员密码）。

---

### V-003：硬编码弱密钥 🔴

**位置：** `app.py` 第 5 行

**问题代码：**
```python
app.secret_key = "dev-key-2025"
```

**风险：**
- 密钥强度极低，可被暴力破解
- session 可被伪造，导致任意用户登录
- 一旦泄露，所有用户的 session 都可被解密

**利用方式：** 攻击者知道密钥后，可伪造任意用户的 session cookie，无需密码即可登录任何账号。

## 3. 中危漏洞详解

### V-004：调试信息泄露 🟠

**位置：** `templates/login.html` 第 1 行

**问题代码：**
```html
<!-- 调试信息 - 默认管理员账号 用户名: admin 密码: admin123 -->
```

**风险：**
- 查看网页源码即可获得管理员账号密码
- 即使密码改了，这个注释若没删就是定时炸弹

**利用方式：** 攻击者浏览器查看页面源码 → 直接获取管理员凭证。

---

### V-005：无会话安全配置 🟠

**位置：** `app.py` 第 5 行

**问题代码：**
```python
app.secret_key = "dev-key-2025"
# 缺少 session 安全配置
```

**缺失配置：**
```python
app.config['SESSION_COOKIE_HTTPONLY'] = True     # 防止 JS 读取 cookie
app.config['SESSION_COOKIE_SAMESITE'] = 'Lax'    # 防止 CSRF
app.config['SESSION_COOKIE_SECURE'] = True        # 仅 HTTPS 传输
app.config['SESSION_PERMANENT'] = True
app.config['PERMANENT_SESSION_LIFETIME'] = ...     # session 过期时间
```

**风险：** session cookie 可通过 XSS 窃取，容易被 CSRF 攻击利用。

---

### V-006：无 CSRF 保护 🟠

**位置：** `templates/login.html` 登录表单

**问题代码：** 无 `{{ csrf_token() }}`

**风险：** 攻击者可构造恶意页面，诱导用户提交表单（跨站请求伪造）。

**利用方式：**
```html
<!-- 攻击者构造的恶意页面 -->
<form action="http://目标/login" method="POST">
  <input name="username" value="admin">
  <input name="password" value="123456">
</form>
<script>document.forms[0].submit()</script>
```

---

### V-007：暴力破解防护缺失 🟠

**位置：** `app.py` 第 35-46 行，login 路由

**问题代码：**
```python
@app.route("/login", methods=["GET", "POST"])
def login():
    if request.method == "POST":
        # 无频率限制，可无限尝试密码
```

**风险：**
- 攻击者可通过脚本无限尝试密码
- 无验证码、无锁定机制

---

### V-008：开启调试模式 🟠

**代码：**
```python
app.run(debug=True, host="0.0.0.0", port=5000)
```

**风险：**
- debug=True 会暴露 Werkzeug 调试器
- 调试器可执行任意 Python 代码
- 错误页面会显示完整的调用栈和源代码片段

## 6. 修复后核心代码

### 6.1 密码哈希处理

```python
from werkzeug.security import generate_password_hash, check_password_hash

# 密码哈希存储
USERS = {
    "admin": {
        "username": "admin",
        "password": generate_password_hash("admin123"),  # 哈希存储
        "role": "admin",
        "email": "admin@example.com",
        "phone": "13800138000",
        "balance": 99999
    },
    "alice": {
        "username": "alice",
        "password": generate_password_hash("alice2025"),
        "role": "user",
        "email": "alice@example.com",
        "phone": "13900139001",
        "balance": 100
    }
}

# 登录验证
if user and check_password_hash(user["password"], password):
    # 密码正确
```

### 6.2 安全密钥管理

```python
import os

app.secret_key = os.environ.get('SECRET_KEY', os.urandom(32).hex())
# 优先从环境变量读取，否则随机生成
```

### 6.3 Session 安全配置

```python
app.config.update(
    SESSION_COOKIE_HTTPONLY=True,
    SESSION_COOKIE_SAMESITE='Lax',
    SESSION_COOKIE_SECURE=True,      # HTTPS 环境
    SESSION_PERMANENT=True,
    PERMANENT_SESSION_LIFETIME=timedelta(hours=2),
)
```

### 6.4 密码不返回前端

```python
@app.route("/")
def index():
    username = session.get("username")
    user = USERS.get(username)
    if user:
        # 复制用户信息但不包含密码字段
        safe_user = {k: v for k, v in user.items() if k != 'password'}
        return render_template("index.html", user=safe_user)
    return render_template("index.html", user=None)
```

### 6.5 错误信息规范化

```python
# 不要明确告诉用户"用户名或密码错误"
# 统一返回："认证失败"
return render_template("login.html", error="用户名或密码错误")
# 改为：
return render_template("login.html", error="认证失败，请检查您的凭证")
```
